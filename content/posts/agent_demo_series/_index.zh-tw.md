## 為什麼要自建 coding agent

現在 claude code, codex 等能寫程式的強力 agent 滿天飛，為什麼還要自己建立 coding agent？

這個系列誕生的起因是因為在 2025 年的 DevOpsDays 上聽到
[安德魯架構師](https://columns.chicken-house.net/) 提到，身為軟體工程師，應該要知道像 cursor 這類的 coding agent 是怎麼做的。當下聽到發現自己的確對於 agent 還沒有過深入的了解，而這個種子埋到了現在（好像隔有點久），終於要來把自己缺失的相關知識補上。

另外這個系列的出發點不是真的要去建立一個**生產級**的專案，而是想要藉由建立 coding agent 的過程中，去理解一個 agent 背後可能涉及到的技術細節與架構。且是從我自己的角度出發的，所以除了 agent 本身的相關原理之外，還會針對我自己沒有那麼熟，也想一起深挖的相關知識，甚至也額外包含了一部分跟 AI 協作的相關流程等等。所以雖然開頭是說不要做一個生產級，但是製作的中可一點都不馬虎😎。BDD 與 TDD 都用上了呢！
